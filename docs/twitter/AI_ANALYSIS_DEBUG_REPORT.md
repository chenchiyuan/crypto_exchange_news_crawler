**为什么0️⃣多空一致性统计都是0？**

**原因**：
1. AI在"即时信号流"中正确识别了观点（3条信号，2条看空，1条看多）
2. 但在"0️⃣多空一致性统计"中，所有数据都是0
3. 这说明AI没有正确将"即时信号流"中的观点计入"多空一致性统计"

**可能的错误**：
- AI没有理解提示词的逻辑要求："观点统计——最近7天"
- AI认为这3条信号不足以形成统计（只有3条推文有明确观点，其他12条都是无观点内容）
- AI可能过度保守，只在观点非常明确时才计入统计

### 2. 推送格式化问题

**推送内容显示**：
```
⚡ 3️⃣ 即时信号流：
  • N/A — @@NachoTrades: N/A N/A (置信度: N/A)
  • N/A — @@RunnerXBT: N/A N/A (置信度: N/A)
  • N/A — @@RunnerXBT: N/A N/A (置信度: N/A)
```

**问题**：时间、方向、资产都显示N/A

**原因**：推送格式化代码中，字段名映射错误：
- `signal.get('time', 'N/A')` - 实际字段是 `signal.get('时间', 'N/A')`
- `signal.get('direction', 'N/A')` - 实际字段是 `signal.get('方向', 'N/A')`
- `signal.get('asset', 'N/A')` - 实际字段是 `signal.get('资产', 'N/A')`

## 🛠️ 解决方案

### 方案1：修复推送格式化
在 `twitter/services/notifier.py` 中修复字段名映射，使用中文键名

### 方案2：优化AI提示词
在提示词中更明确地要求：
- 即使观点数量少，也要计入统计
- 区分"无观点"和"观点不足"两种情况
- 明确说明对于小样本数据的处理方式

### 方案3：改进分析逻辑
在AI分析服务中增加后处理逻辑：
- 将"即时信号流"中的观点同步到"多空一致性统计"
- 确保数据一致性
