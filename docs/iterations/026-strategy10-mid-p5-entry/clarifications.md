## 澄清问题与决策

### Q1: 入场条件中的"mid"指什么？

**问题**: 用户描述中的"mid"具体指哪个指标？

**决策**:
- **mid = inertia_mid（惯性中值）**
- 惯性中值是基于β斜率和EMA25计算的趋势中心位置
- 公式：`inertia_mid = EMA25 + (β × T_adj / 2)`

**依据**: 在现有策略体系中，"mid"通常指惯性中值，与P5/P95配合使用。

---

### Q2: 入场条件是"大于等于"还是"小于等于"？

**问题**: 用户描述"close价格大于等于（p5+mid）/ 2"，但通常做多策略是低位买入。

**分析**:
- 传统做多策略：价格下跌到支撑位时买入（如策略7: low <= P5）
- 用户需求：close >= (P5 + mid) / 2

**决策**:
- **严格按用户需求实现**：`close >= (P5 + mid) / 2`
- 这是一种"确认反弹"的入场策略，而非"抄底"策略
- 当价格从低位回升到中值和P5的中点时，认为反弹确认，入场做多

---

### Q3: 0.3%止盈是基于什么价格？

**问题**: 0.3%利润止盈的计算基准是什么？

**决策**:
- **基于开仓价格的浮动盈利**
- 公式：`(current_close - open_price) / open_price >= 0.003`
- 当浮动盈利达到0.3%时触发止盈

---

### Q4: 两个出场条件的优先级？

**问题**: EMA25回归和0.3%止盈同时满足时如何处理？

**决策**:
- **先到先触发原则**
- 在同一根K线上，两个条件可能同时满足
- 按配置顺序检查，先满足的先触发
- 实际上两个条件触发时机通常不同，不太可能同时

---

### Q5: 使用close还是low/high判断入场？

**问题**: 入场条件是判断close还是判断low/high？

**决策**:
- **使用close（收盘价）**
- 用户明确指定"close价格"
- 这与策略7（low <= P5）不同
